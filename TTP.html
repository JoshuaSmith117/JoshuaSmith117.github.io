Learning from Play-Testing


When we created the first prototype of TapTapPop!, we weren't concerned with a polished user interface, we simply needed a version that displayed everything the player might need to know. It displayed a current score count, lives and highscore along the top edge of the screen. The display used basic black font.
While this format technically displayed what the player needed to see, it not only looked thrown together, but we learned through play-testing that it had two problems: it either distracted the player from the rather hectic gameplay, or the player never saw it because they were too focused on the gameplay. These problems ultimately hurt the player’s score, and more importantly, their experience.


Making Effective UI Changes Based on Feedback


In order to address these issues, I first decided that displaying the highscore during gameplay was unnecessary, as we could simply display it on the game over screen, so I removed it. Next, I merged the "Lives" and "Score" values into a single entity and moved them into the center of the screen in the background. The score is displayed clearly in the center of the screen, but since it’s grayed out and in the background, it doesn’t distract the eye. But where are the Lives being displayed? The Lives are actually displayed based on the size of the score. Whenever a life is lost, the number gets significantly larger. This was confusing to the player for the first couple tries, but since the gameplay is very short and retrying occurs a lot, they learned quickly what this size change meant. During later playtesting, there were no more complaints about distraction.


Creating Simple, Mobile-Friendly Menus


When creating menus for a mobile app, less is always better. People play mobile games with just a few minutes to spare, so they need to be able to get right into the action. This is the philosophy that drove my menu design for TapTapPop!
In TapTapPop! we decided that the only necessary options were centered around the adjusting of audio. Unlike a PC game, mobile games don't usually require any display options. This minimal options menu allowed us to easily and fit our options directly onto the main menu screen, preventing the player from having to jump through extra, unnecessary hoops to change the audio.
Similar to the main menu, the game over screen is very minimal in nature. It displays only the basic necessities: score, highscore, a retry button and a main menu button. Notice how the retry button is larger than the main menu button and is closer to the center of the screen. This causes players to be more likely to press it than if it shared equal weight with the main menu button.

Introduction


The Mark Cuban Center for Sports Media and Technology at IU needed a small online arcade of IU Athletics games to be built. The primary purpose of this arcade was to gather email addresses so that the Mark Cuban Center could then advertise tickets for upcoming sporting events to these email addresses. I designed and developed the website that would house these games and make it easy and for users to play them. I even created the login system and created incentive to register. I also led development and designed the user experiences for the games themselves. Learn more about my game design ****here.


Utilizing Responsive Design Techniques


Today, it is required that all websites are responsive and scale to any screen size. With more people browsing the web on their smartphones than ever before, I created a website that functions properly across all devices.

All elements of each page fit within a body div element that has a max-width of around 1000 pixels. This prevents the pages from having too scale proportionately to potentially massive screens. Once the screen size is scaled down to 1000 pixels, it breaks the PC-Mobile threshold. At this point, the screen will switch to a mobile format and elements will begin scaling down accordingly. I do this scaling in a series of steps. For example, at 720 pixels or less, all text elements shrink to a smaller font size. Game window sizes shrink accordingly as well.

In addition to element downsizing, unnecessary elements, such as text in the login bar, will disappear entirely upon reaching a small enough screen width. The body’s width also runs flush with the edges of the screen upon entering the mobile format, allowing it and all inner div elements to scale down along with the screen width. Many of the elements even align to the center of the screen while in the mobile format.


Persuading Users to Register


The primary goal of the Mark Cuban Center with this website is to provide a fun gaming experience, but also to gather user emails so they can then email users with ticket sales for upcoming sporting events. To accomplish this task, I developed a system for saving the users’ high-scores to a database, but only while they are logged in.

This high-score system acts as some good incentive to login, however players need to be well-aware of the system so that they can take advantage of it. On every page, along with each game link, there is an area that displays the user’s high-score for that game. In each of those areas, the user will see: “Highscore: Please Login”. This acts as a constant reminder, no matter where the user is on the site, that logging in will allow them to see their high-score.

To add to this, throughout every page, I placed the login bar at the top-center of the screen with brightly colored buttons to highlight it. I even added text to the login bar that guides the user toward a call to action. “First time here? Register. Already Registered? Login.” This text is context sensitive as well. If the player is on the game page, playing a game, the text instead asks: “Want to save your highscores for this game? Login or Register.” This helpful and person-like text might just encourage the user to take that extra step toward registering.


Case Study: The Home Page


The home page is arguably the most important page for any website. It’s the first thing that a user sees when entering a site, so it must make a good impression and encourage them to explore the site further by making it easy for them to find what they’re looking for.

The first impression that I wanted users to get from this sight is this: It’s a high energy arcade with a lot of potential for fun. To convey this message as soon as possible, I want users to immediately see a lot of movement. I took inspiration from IU’s main home page and included a full-width video displays on the screen as soon as the user enters the site. This creates a lot of movement, while also showing them a sneak peak of the games that they could play in this arcade.

Users also need to be reminded where they are within the first moments of visiting the site. They will be expecting an IU site, so they need to be met by a warm IU greeting. This is handled by the large header across the top and the logo directly in the center of the video. The red and white color scheme also adds to the IU theming.

After the first few moments, the user will get to inspect the less impactful areas of the page. They might spot the brightly colored login buttons (one of the most important aspects of the site) and the social media icons in the top right corner (The Mark Cuban Center cares about their social media outreach).

Soon, the user will scroll down to see the full list of games that they can play, clearly presented down the center of the screen. Each game is presented with a thumbnail image, title, short description, and high-score display. This information helps the user make an informed decision of which game they want to play, if any. If they hover over the thumbnail images or the title text, a red highlight will grab their attention and let them know that these game sections are clickable. By selecting a game from this list, the player will be sent directly to the game page. In just one click, they’ve gone from entering the site to being able to play.

Finally, in a small footer at the bottom of the screen, the user will find the legal and contact information for the Mark Cuban Center. None of this information is at the top of the Mark Cuban Center’s priority list. It’s available to anyone who wants to view it, but it’s neatly out of the way for anyone who doesn’t care and just wants to jump into the action.


Case Study: The Game Page


The game page is another important page on the site. This is where the user can dive right in and play the games. It looks very similar to the home page, with the same header, login bar, games list, and footer. However, just under the header and login bar, I have placed the game window. This is where the action happens. Alongside the game, sits a full description of the game as well as the user’s high score. If they are not logged in, the high-score area instead asks that they please log in, further working toward persuading the user to log in.

Unfortunately, certain games have a loading time of a few seconds. This causes the game to display a black screen for the first few seconds after the page opens, which may lead users to believe that the page isn’t working properly. I attempted to remedy this issue by reminding users that the games may take a few seconds to load. It’s a small addition, but it can make a large impact on the experience of the user by keeping them on the page for just a few more seconds while the game loads.

We still had the issue of text readability, however, and to solve this problem, I simply changed the text font to be less stylized and easier to render with fewer pixels.
